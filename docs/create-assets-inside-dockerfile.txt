Goal
----
Compile JS/CSS assets inside the Docker image build (multi-stage Dockerfile)
so that the generated app.js and app.css no longer need to be committed to git.

Background
----------
asset-builder is a Rack/Sinatra app that serves compiled Sprockets assets over
HTTP. build_assets.sh currently spins it up, curls the compiled files into
source/public/assets/, and those files are committed to git.

A compile.sh was added to asset-builder (image tag f2bcab7) that compiles the
same assets in one shot without starting a server:

  /app/config/compile.sh <out-dir>

This enables a multi-stage Dockerfile build.

Constraint
----------
The sinatra-base CI creates automated PRs that update line 1 of the web
Dockerfile.  Currently line 1 is a bare FROM with no alias:

  FROM ghcr.io/cyber-dojo/sinatra-base:TAG@sha256:HASH

A three-stage Dockerfile is needed:

  Stage 1  sinatra-base, aliased AS base  (line 1 - automated PR target)
  Stage 2  asset_builder AS assets        (compiles JS/CSS)
  Stage 3  FROM base                      (final image, copies compiled assets)

For this to work, the sinatra-base automated PR must write:

  FROM ghcr.io/cyber-dojo/sinatra-base:TAG@sha256:HASH AS base

instead of the current bare FROM.

Steps
-----
1. sinatra-base repo
   Update the automated PR script so it appends "AS base" when it rewrites
   line 1 of downstream Dockerfiles.

2. web repo - Dockerfile
   After step 1 is live, rewrite to three stages:

     FROM ghcr.io/cyber-dojo/sinatra-base:... AS base

     FROM cyberdojo/asset_builder:f2bcab7 AS assets
     COPY source/app/assets/javascripts /app/app/assets/javascripts
     COPY source/app/assets/stylesheets /app/app/assets/stylesheets
     RUN /app/config/compile.sh /out

     FROM base
     LABEL maintainer=jon@jaggersoft.com
     ARG COMMIT_SHA
     ENV SHA=${COMMIT_SHA}
     ARG APP_DIR=/web
     ENV APP_DIR=${APP_DIR}
     WORKDIR ${APP_DIR}/source
     COPY source/ .
     COPY --from=assets /out/app.js  public/assets/app.js
     COPY --from=assets /out/app.css public/assets/app.css
     USER nobody
     HEALTHCHECK --interval=1s --timeout=1s --retries=5 --start-period=5s CMD ./healthcheck.sh
     ENTRYPOINT ["/sbin/tini", "-g", "--"]
     CMD [ "./up.sh" ]

3. web repo - Makefile
   Remove the "image: assets" dependency. The assets target remains for local
   development (make assets still works via the HTTP server approach).

4. web repo - .gitignore
   Add:
     /source/public/assets/app.js
     /source/public/assets/app.css

5. web repo - git
   Remove the compiled files from tracking:
     git rm --cached source/public/assets/app.js
     git rm --cached source/public/assets/app.css

Note: bin/echo_env_vars.sh still references CYBER_DOJO_ASSET_BUILDER_TAG for
the local "make assets" dev flow. That tag should be updated to f2bcab7 and
kept in sync with the tag used in the Dockerfile (either manually or by a
second automated PR from the asset_builder CI).
